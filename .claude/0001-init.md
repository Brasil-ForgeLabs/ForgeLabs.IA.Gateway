## Project Goals

The project is a Modular Monolith built with .NET 8.

Current solution structure:

src/
│
├── Api
│
├── BuildingBlocks
│   ├── Application
│   ├── Domain
│   └── Infrastructure
│
├── Modules
│   ├── Inference
│   ├── Metrics
│   └── Shared
│
└── Tests

The BuildingBlocks project contains only reusable technical capabilities shared by every module.

Examples:

Application
- CQRS abstractions
- Mediator abstractions
- Validation
- Pipeline Behaviors

Domain
- Entity
- AggregateRoot
- ValueObject
- Domain Events
- Base abstractions

Infrastructure
- Database
- Redis
- Messaging
- HTTP
- Email
- Blob Storage
- External integrations

Business-specific abstractions should NEVER live inside BuildingBlocks.

Each module owns its own domain.

## Current Focus

I want to start implementing only the Inference module.

At this moment the Inference module has only one responsibility:

Receive an inference request.

Select the correct AI provider.

Transform the request into the provider format.

Execute the request.

Normalize the response.

Return the response.

It should NOT:

- Calculate metrics
- Generate reports
- Calculate billing
- Manage authentication
- Cache responses
- Handle dashboards

These responsibilities belong to other modules.

---

## Future Vision

Later I want the gateway to expose chat endpoints with persistent conversations.

For example:

Create Session

Add Messages

Retrieve Conversation History

Continue Conversation

However, I am not sure if:

Conversation should belong to the Inference module

or

Conversation should become its own independent module that internally uses the Inference module.

I want you to analyze this decision.
